# Phase 0: Ideation & Research

> **Tools:** AI Chat (Gemini / ChatGPT / Claude), Web Search, Notion
> **Duration:** 1–3 hours
> **Output:** Clear problem statement + rough feature list saved as `docs/00-ideation.md`

## Steps

1. **Identify the Problem** — What pain point does this solve? For whom? Write it in 1–2 sentences.
2. **Research Competitors** — Find 3–5 competitors or similar apps. Note:
   - What they do well
   - What's missing or annoying
   - Their pricing model (includes average fullstack developer or software developer to build this kind of project)
3. **Define Target Audience** — Who will use this? Be specific (age, tech-savviness, location, budget).
4. **Brainstorm USPs** — What makes YOUR solution different from competitors?
5. **Outline Core Features** — List 5–10 MUST-HAVE features for MVP. Create a separate "Nice to Have" backlog.
6. **Choose a Name** — Simple, memorable, relevant. Check domain availability.
7. **Validate** — Ask: would someone actually pay for this? Is the market big enough?

## AI Prompt Template

```
Act as a Senior Product Manager. I want to build [type of app] that solves [problem].

Target audience: [who]
Competitors: [list if known]

Help me:
1. Validate this idea — is it worth building?
2. Define the MVP features (maximum 8)
3. Suggest a catchy name
4. Identify potential revenue model
5. Outline technical risks or challenges
```

## Output

Save the result as `docs/00-ideation.md` with these sections:
- Problem Statement
- Target Audience
- Feature List (MVP vs Future)
- Competitive Analysis (table format)
- Unique Selling Points
- Name Candidates
- Revenue Model Ideas
